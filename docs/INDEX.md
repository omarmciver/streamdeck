## Documentation Standards

### Document Structure

**All documents follow:**
1. **Title & Overview** - What this document covers
2. **Table of Contents** (for long documents)
3. **Main Content** - Organized by topic/subtopic
4. **Examples** - Code samples and usage patterns
5. **Cross-References** - Links to related documentation
6. **Quick Reference** (where applicable)

### Code Examples

**All code examples include:**
- File path references (e.g., `src/plugin.ts:1-13`)
- Language syntax highlighting
- Inline comments for clarity
- Context about when to use the pattern

### Cross-Referencing

**Format:** `[Link Text](filename.md#section)`

**Examples:**
- `[ARCHITECTURE.md](ARCHITECTURE.md)` - Link to document
- `[Event Lifecycle](ARCHITECTURE.md#event-lifecycle)` - Link to section
- `[README.md → Installation](../README.md#installation)` - External + section
